## The config file

Kyaraben stores its configuration at `~/.config/kyaraben/config.toml`. The
config file and the UI are kept in sync: changes made in the UI are written to
the file, and changes made to the file are reflected in the UI.

Most users do not need to edit the config file directly. It is there for users
who prefer text editors or want to version-control their setup.

### Example

```toml
[global]
user_store = "~/Emulation"

[systems]
snes = ["retroarch:bsnes"]
psx = ["duckstation"]
gamecube = ["dolphin"]
ps2 = ["pcsx2"]

[emulators.duckstation]
version = "v0.1-10655"

[frontends.esde]
enabled = true

[sync]
enabled = true
mode = "primary"
```

### `[global]`

| Key          | Type   | Default       | Description                     |
| ------------ | ------ | ------------- | ------------------------------- |
| `user_store` | string | `~/Emulation` | Path to the emulation directory |

### `[systems]`

Maps system IDs to lists of emulator IDs. Each system can have one or more
emulators enabled.

```toml
[systems]
snes = ["retroarch:bsnes"]
psx = ["duckstation"]
```

### `[emulators.<id>]`

Optional per-emulator configuration. Currently supports version pinning:

```toml
[emulators.duckstation]
version = "v0.1-10655"
```

When no version is specified, kyaraben uses the latest supported version.

### `[frontends.<id>]`

Frontend configuration:

```toml
[frontends.esde]
enabled = true
```

### `[sync]`

See [save sync](/sync/) for sync configuration.

## Commands

The CLI is intentionally minimal. The config file is the interface for
advanced users. The UI is for users who prefer not to edit files. The CLI is
the glue: initialize, apply, check status.

### `kyaraben apply`

Reads the config, installs emulators, generates emulator configs, and creates
the folder structure. If there are conflicts with your existing configuration,
`kyaraben apply` asks for confirmation before overwriting.

```
$ kyaraben apply

Applying kyaraben configuration...

Using ~/Emulation (existing data preserved)
Installing emulators (this may take a while on first run)...
Adding to application menu...
Configuring emulators...
  Installed to: ~/.local/state/kyaraben/packages/...

  Applied: ~/.config/retroarch/retroarch.cfg
  Applied: ~/.config/duckstation/settings.ini
  Applied: ~/.config/mgba/config.ini
  Applied: ~/.config/PCSX2/inis/PCSX2.ini

Done!

Your emulation directory is ready at: ~/Emulation
Place your ROMs in the appropriate subdirectories.
```

Use `--dry-run` to preview config changes without applying them.

### `kyaraben init`

Creates a default `config.toml` with popular systems enabled:

```
$ kyaraben init

Created configuration at ~/.config/kyaraben/config.toml

Enabled 13 systems with default emulators.
Run 'kyaraben apply' to install emulators and create directories.
```

### `kyaraben uninstall`

Removes kyaraben-managed files. Never touches the <UserStore />.

```
$ kyaraben uninstall

  ~/.local/state/kyaraben (packages, manifest, state)

  Kyaraben installation:
    ~/.local/bin/kyaraben-ui
    ~/.local/bin/kyaraben
    ~/.local/share/applications/kyaraben.desktop

  Desktop entries:
    ~/.local/share/applications/kyaraben/duckstation.desktop
    ~/.local/share/applications/kyaraben/retroarch.desktop
    ~/.local/share/applications/kyaraben/mgba.desktop

  Icons:
    ~/.local/share/icons/hicolor/256x256/apps/kyaraben-duckstation.png

```

### `kyaraben sync`

Manage save synchronization. See [save sync](/sync/).

#### `kyaraben sync pair [CODE]`

Pair two devices. Without a code, this device becomes the primary and
displays a pairing code. With a code, this device becomes the secondary and
joins the primary that generated the code.

Installs and starts the sync service automatically if it is not already
running.

```
$ kyaraben sync pair
Installing sync service...
Pairing code: K7X9M2
Waiting for devices... (expires in 5 minutes)

Paired with steamdeck-kyaraben
```

#### `kyaraben sync status`

Shows sync state, connected devices, and per-folder progress.

```
$ kyaraben sync status

Sync: active (primary)
Device ID: LGFPDIT7-SKNNJVJZ-A4FC7QNC-...

Devices:
  steamdeck-kyaraben    connected    last seen: just now

Folders:
  ROMs (6 systems)      synced       12.3 GB
  Saves (6 systems)     synced       156 MB
  States (4 systems)    synced       1.2 GB
  Screenshots           synced       89 MB
```

#### `kyaraben sync pause` / `kyaraben sync resume`

Pause or resume sync. Useful on metered connections or to save battery.

#### `kyaraben sync add-device <DEVICE-ID>`

Add a device by its 56-character Syncthing device ID. Use this when devices
are not on the same network and mDNS discovery is not available.

#### `kyaraben sync remove-device <ID-or-NAME>`

Remove a paired device by its device ID or friendly name.

#### `kyaraben sync resolve`

List and resolve sync conflicts interactively.
