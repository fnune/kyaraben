## Overview

Kyaraben uses Syncthing to synchronize emulation data across devices. Syncthing
is installed automatically when you enable sync. No accounts, cloud services,
or port forwarding required.

## Enabling sync

### Using the app

1. Go to Settings > Sync
2. Choose Primary or Secondary mode
3. Click "Enable sync"

**Primary**: the device where your ROM collection lives (typically a desktop or
server). Sends ROMs and BIOS to secondaries.

**Secondary**: receives ROMs from the primary and syncs saves bidirectionally
(typically a Steam Deck or laptop).

### Using the CLI

Edit `~/.config/kyaraben/config.toml`:

```toml
[sync]
enabled = true
mode = "primary"  # or "secondary"
```

Then run `kyaraben apply` to install Syncthing.

## Pairing devices

### On the same network

On the primary:

1. Go to Settings > Sync
2. Click "Start pairing"
3. A 6-character code appears

On the secondary:

1. Go to Settings > Sync
2. Enter the code from the primary
3. Click "Join primary"

The secondary discovers the primary automatically via mDNS. Pairing completes
in seconds.

### On different networks

When devices are not on the same local network, exchange device IDs manually:

```
# On each device, get the device ID
$ kyaraben sync status
Device ID: LGFPDIT7-SKNNJVJZ-A4FC7QNC-...

# On each device, add the other device's ID
$ kyaraben sync add-device <DEVICE-ID-FROM-OTHER-DEVICE>
```

Syncthing's relay servers handle connectivity. This works across any network
but transfers are slower than a direct connection.

## What syncs

| Content        | Primary       | Secondary     |
| -------------- | ------------- | ------------- |
| `roms/`        | Send only     | Receive only  |
| `bios/`        | Send only     | Receive only  |
| `saves/`       | Bidirectional | Bidirectional |
| `states/`      | Bidirectional | Bidirectional |
| `screenshots/` | Bidirectional | Bidirectional |

ROMs and BIOS flow one direction only, preventing accidental deletion of your
primary collection from a secondary device.

## Managing sync

### Checking status

```
$ kyaraben sync status

Sync: enabled (primary)
Device ID: LGFPDIT7-SKNNJVJZ-A4FC7QNC-...

Paired devices:
  steamdeck-kyaraben    connected
```

### Removing a device

In the app, click the trash icon next to the device. From the CLI:

```
$ kyaraben sync remove-device steamdeck-kyaraben
```

## Configuration

Syncthing runs as a systemd user service with its own configuration, separate
from any existing Syncthing installation.

| Setting          | Default                              |
| ---------------- | ------------------------------------ |
| Config directory | `~/.local/state/kyaraben/syncthing/` |
| Listen port      | 22100                                |
| Discovery port   | 21127                                |
| GUI port         | 8484                                 |
| Relay servers    | Enabled                              |
| Device name      | `<hostname>-kyaraben`                |

Override ports in `~/.config/kyaraben/config.toml`:

```toml
[sync.syncthing]
listen_port = 22100
discovery_port = 21127
gui_port = 8484
relay_enabled = true
```

### Ignore patterns

Kyaraben ignores regenerable data by default:

```
**/shader_cache/**
**/cache/**
**/*.tmp
.DS_Store
Thumbs.db
```

Add custom patterns:

```toml
[sync.ignore]
patterns = [
  "**/shader_cache/**",
  "**/logs/**",
]
```

### File versioning

Syncthing keeps old versions of save and state files for 30 days. ROM and BIOS
folders do not use versioning.

## Troubleshooting

### Pairing fails with "No devices found"

mDNS discovery requires both devices on the same local network. Some networks
block mDNS:

- Guest WiFi networks isolate clients
- Routers with "AP isolation" enabled
- Corporate networks blocking multicast

Use manual device ID exchange instead.

### Pairing code rejected

Codes expire after 5 minutes. Cancel pairing on the primary and start a new
session.

### Devices paired but not syncing

1. Check `kyaraben sync status` shows "connected"
2. Verify no firewall blocks ports 22100 (TCP) and 21127 (UDP)
3. If on different networks, ensure relay servers are enabled

### SteamOS firewall

The Steam Deck may block Syncthing ports. Allow them:

```
$ sudo firewall-cmd --zone=public --add-port=22100/tcp --permanent
$ sudo firewall-cmd --zone=public --add-port=21127/udp --permanent
$ sudo firewall-cmd --reload
```
