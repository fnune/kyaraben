## Overview

Kyaraben uses Syncthing to synchronize emulation state across devices. A
bundled Syncthing instance handles the file transfer. You do not need to
install Syncthing separately.

## Roles

Each device operates as either `primary` or `secondary`:

- A `primary` creates all system folders and shares them. It is typically a
  media server or main PC where your ROM collection lives.
- A `secondary` subscribes to folders for its enabled systems and installs the
  corresponding emulators. It is typically a Steam Deck or laptop.

## Sync direction

| Content        | Primary       | Secondary     |
| -------------- | ------------- | ------------- |
| `roms/`        | Send only     | Receive only  |
| `bios/`        | Send only     | Receive only  |
| `saves/`       | Bidirectional | Bidirectional |
| `states/`      | Bidirectional | Bidirectional |
| `screenshots/` | Bidirectional | Bidirectional |
| `opaque/`      | Bidirectional | Bidirectional |

ROMs and BIOS files flow one direction: from primary to secondaries. This
prevents accidental deletion of your primary ROM collection from a secondary
device.

Saves and states sync both ways. Play on your Steam Deck, continue on your
desktop.

## Setting up sync

### On the primary device

Set the sync mode to `primary` in the config:

```toml
[sync]
enabled = true
mode = "primary"
```

Or enable sync through the UI settings.

### Pairing devices

Use the CLI or UI to pair devices:

```
$ kyaraben sync add-device
```

The primary auto-accepts folder shares from paired devices.

### On secondary devices

Set the sync mode to `secondary`:

```toml
[sync]
enabled = true
mode = "secondary"
```

Enable the systems you want to play. Kyaraben creates the local folders and
subscribes to the corresponding shared folders from the primary.

## Bundled Syncthing

Kyaraben ships its own Syncthing instance with separate configuration to avoid
conflicts with any existing Syncthing installation.

| Setting          | Default                         |
| ---------------- | ------------------------------- |
| Config directory | `~/.config/kyaraben/syncthing/` |
| Listen port      | 22001                           |
| Discovery port   | 21028                           |
| GUI port         | 8385                            |
| Relay servers    | Enabled                         |
| Device name      | `<hostname>-kyaraben`           |

All ports are configurable in `config.toml`:

```toml
[sync.syncthing]
listen_port = 22001
discovery_port = 21028
gui_port = 8385
relay_enabled = true
```

## File versioning

Syncthing keeps old versions of files in `saves/`, `states/`, and `opaque/`
directories. Versions are kept for 30 days with staggered cleanup. This
protects against accidental overwrites or corruption.

## Ignore patterns

Kyaraben configures Syncthing to ignore regenerable data:

```
**/shader_cache/**
**/cache/**
**/*.tmp
.DS_Store
Thumbs.db
```

Custom ignore patterns can be added in the config:

```toml
[sync]
ignore_patterns = [
  "**/shader_cache/**",
  "**/logs/**",
]
```
