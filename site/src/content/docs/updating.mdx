## Updating kyaraben

:::caution[Work in progress]
Automatic updates for kyaraben itself are planned but not yet implemented.
:::

To update kyaraben, download the latest AppImage from the
[GitHub releases page](https://github.com/fnune/kyaraben/releases) and replace
the existing one.

The planned auto-update mechanism will allow kyaraben to check for new versions
and update itself. The AppImage format supports this by downloading a new
version and replacing the current binary.

## Updating emulators

When emulator updates are available, kyaraben can show what will change:

```
$ kyaraben update

Checking for updates...

Updates available:
  RetroArch      1.18.0  →  1.19.1
  DuckStation    0.1-6722  →  0.1-10655

Apply updates? [y/N]
```

:::caution[Work in progress]
The update command is under development.
:::

## Version pinning

You can pin a specific emulator version in your config to prevent automatic
updates:

```toml
[emulators.duckstation]
version = "v0.1-10655"
```

When a version is pinned, kyaraben uses that exact version regardless of
whether newer versions are available. Remove the version key or set it to
`"latest"` to follow updates.
