## Download

Download the latest AppImage from the
[GitHub releases page](https://github.com/fnune/kyaraben/releases) and double-click it to execute it.

Kyaraben runs on any Linux distribution, including SteamOS for the Steam Deck.
To install it on the Steam Deck, press the `STEAM` button, select Power, and
choose Switch to Desktop.

If you're comfortable using the terminal:

```bash
chmod +x Kyaraben-x86_64.AppImage
./Kyaraben-x86_64.AppImage
```

:::note
x86_64 is fully supported. ARM64 devices have partial support: some emulators
(Eden, DuckStation, PPSSPP, Dolphin) have ARM builds, but most do not. If you
run into issues on your distribution, please
[report them](https://github.com/fnune/kyaraben/issues).
:::

## First run

Kyaraben asks where to create your collection. The default is
`~/Emulation`. This is where all your ROMs, BIOS files, saves, and screenshots
live.

<img src={welcomeScreenshot.src} alt="Welcome screen showing the collection picker at the top" />

## Selecting systems

The main screen shows available gaming systems grouped by manufacturer. Check
the systems you want to emulate.

Each system has a default emulator chosen for the best out-of-the-box
experience. The UI shows which emulators are available for each system.

<img
  src={systemPickerScreenshot.src}
  alt="System picker with Nintendo systems enabled and pending changes at the bottom"
/>

## Applying changes

Click apply. Kyaraben fetches the emulators, generates their configuration
files, and creates the folder structure inside your collection. On
first apply, Kyaraben also installs itself to your system, adding `kyaraben`
and `kyaraben-ui` to your PATH and creating a desktop entry so you can launch
it from your application menu. If you prefer a headless experience, the CLI has
full feature parity with the UI.

<img
  src={applyProgressScreenshot.src}
  alt="Apply progress showing emulators being downloaded and installed"
/>

If there are conflicts with your existing configuration, Kyaraben shows exactly
what will be modified. See [app reference](/using-the-app/) for details on the
preflight check.

## Adding ROMs and BIOS files

After applying, your collection contains folders for each enabled
system:

```
~/Emulation/
├── roms/
│   ├── snes/
│   ├── psx/
│   └── ...
├── bios/
│   └── psx/
├── saves/
│   └── (per-system)
├── states/
│   └── (per-emulator)
└── screenshots/
    └── (per-system)
```

Place your ROM files in the appropriate `roms/` subdirectory. For systems that
require BIOS or firmware files, the provisions panel in the UI shows exactly
which files are needed, where to place them, and whether they pass hash
verification. You can also run `kyaraben doctor` from the command line.

## Playing your games

You can launch games in two ways:

- Use [EmulationStation DE](https://es-de.org/) for a unified game browser with
  metadata, artwork, and controller-friendly navigation. Enable it in the
  Kyaraben settings (see [app reference](/using-the-app/#browsing-and-launching-games)).
- Launch emulators directly from your application menu and open ROMs from there.
