# Multi-Emulator Support Design

> **Status: IMPLEMENTED** - This design has been fully implemented. See commits on branch `claude/multi-emulator-support-design-elyQ4`.

This document specifies the design for supporting multiple emulators per system in kyaraben.

## Overview

**Goal**: Allow users to enable multiple emulators for a single system (e.g., DuckStation AND Mednafen for PSX).

**Key decisions**:
- System-centric configuration: users think in systems, not emulators
- `systems.{system}` is always `string[]` (list of emulator IDs)
- Default emulator list populated at config creation time
- Optional `[emulators]` section for version pinning and advanced settings
- External frontends (ES-DE) handle game launching and per-game emulator selection

---

## Configuration Format

### Structure

```toml
[global]
user_store = "~/Emulation"

[systems]
psx = ["duckstation"]                              # Single emulator (default)
ps2 = ["pcsx2"]
switch = ["eden"]
gamecube = ["dolphin"]
wii = ["dolphin"]                                  # Same emulator, different system
n64 = ["retroarch:mupen64plus", "ares"]            # Multiple emulators
gba = ["mgba", "retroarch:mgba"]                   # Multiple emulators
# saturn not listed = not enabled

[emulators]
# Optional: version pinning and advanced config
duckstation = { version = "v0.1-10655" }
dolphin = { version = "v2412" }
# eden uses default version (not listed)

[sync]
enabled = true
```

### Semantics

| Config State | Meaning |
|--------------|---------|
| `psx = ["duckstation"]` | PSX enabled with DuckStation |
| `psx = ["duckstation", "retroarch:mednafen_psx"]` | PSX enabled with both emulators |
| `psx = []` | PSX explicitly disabled |
| `psx` not present | PSX not enabled |

### Config Initialization

When `kyaraben init` creates a new config, it populates systems with their default emulator:

```toml
# Generated by kyaraben init
[systems]
psx = ["duckstation"]
ps2 = ["pcsx2"]
switch = ["eden"]
# ... all systems with their defaults
```

User then:
1. Removes systems they don't want
2. Adds additional emulators to systems that need them
3. Optionally pins versions in `[emulators]`

## Go Types

### Config Model

```go
// internal/model/config.go

type KyarabenConfig struct {
    Global    GlobalConf                  `toml:"global"`
    Systems   map[SystemID][]EmulatorID   `toml:"systems"`            // Changed: []EmulatorID not string
    Emulators map[EmulatorID]EmulatorConf `toml:"emulators,omitempty"` // New section
    Sync      SyncConf                    `toml:"sync,omitempty"`
}

type EmulatorConf struct {
    Version string `toml:"version,omitempty"`
    // Future expansion:
    // IsolatedSaves bool   `toml:"isolated_saves,omitempty"`
    // CustomPaths   Paths  `toml:"paths,omitempty"`
}
```

### Derived State

```go
// EnabledEmulator represents a fully resolved emulator configuration
type EnabledEmulator struct {
    ID       EmulatorID
    Version  string     // Empty = use default
    Systems  []SystemID // Which enabled systems use this emulator
}

// GetEnabledEmulators resolves config into a deduplicated list
func (c *KyarabenConfig) GetEnabledEmulators() []EnabledEmulator {
    // Deduplicate: Dolphin appears once even if enabled for both GC and Wii
    emulatorSystems := make(map[EmulatorID][]SystemID)

    for systemID, emulatorIDs := range c.Systems {
        for _, eid := range emulatorIDs {
            emulatorSystems[eid] = append(emulatorSystems[eid], systemID)
        }
    }

    var result []EnabledEmulator
    for eid, systems := range emulatorSystems {
        version := ""
        if ec, ok := c.Emulators[eid]; ok {
            version = ec.Version
        }
        result = append(result, EnabledEmulator{
            ID:      eid,
            Version: version,
            Systems: systems,
        })
    }

    return result
}
```

### GetConfig Response

Current:
```go

    UserStore string
    Systems   map[SystemID]string  // system → single emulator

```

New:
```go
type GetConfigResponse struct {
    UserStore string
    Systems   map[SystemID][]EmulatorID           // system → emulator list
    Emulators map[EmulatorID]EmulatorConfResponse // emulator → config
}

type EmulatorConfResponse struct {
    Version           string
    ConfiguredVersion string   // What user set (may be empty)
```

Current:
```go

    UserStore string
```

New:
```go
type SetConfigRequest struct {
    UserStore string
    Systems   map[SystemID][]EmulatorID          // system → emulator list
    Emulators map[EmulatorID]EmulatorConfRequest // emulator → config (optional)
```

## UI Changes

### Systems View

**Implemented approach**: Conditional parent/child row pattern

- **Single emulator enabled**: Shows single row with system logo, emulator dropdown, version selector (existing behavior)
- **Multiple emulators enabled**: Shows parent row (system) with child rows (emulators) beneath it

```
┌───────────────────────────────────────────────────────────────┐
│ ☑ │ [PSX Logo]     [BIOS badges]     DuckStation▼  v0.1▼     │  ← Single emulator: compact row
└───────────────────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────────────────┐
│ ☑ │ [PSX Logo]     [BIOS badges]                              │  ← Parent row: system only
├───────────────────────────────────────────────────────────────┤
│ ☑ │     └ DuckStation                    Will install  v0.1▼ │  ← Child row: emulator
├───────────────────────────────────────────────────────────────┤
│ ☑ │     └ RetroArch (Mednafen)           Will install  v1.0▼ │  ← Child row: emulator
└───────────────────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────────────────┐
│ ☐ │ [N64 Logo]                                                │  ← Disabled system
└───────────────────────────────────────────────────────────────┘
```

Key UI behaviors:
- Toggling system checkbox enables/disables entire system (all emulators)
- Child row checkboxes toggle individual emulators
- When all emulators are unchecked, system becomes disabled
- Visual hierarchy with "└" prefix and indentation for child rows
- Child rows have subtle background tint to indicate grouping

### State Management

```typescript
// Current
const [selections, setSelections] = useState<Map<SystemID, EmulatorID>>();

// New
const [systemEmulators, setSystemEmulators] = useState<Map<SystemID, EmulatorID[]>>();
const [emulatorVersions, setEmulatorVersions] = useState<Map<EmulatorID, string>>();

// Helper: toggle emulator for a system
function toggleEmulator(systemId: SystemID, emulatorId: EmulatorID) {
    setSystemEmulators(prev => {
        const current = prev.get(systemId) ?? [];
        const hasEmulator = current.includes(emulatorId);

        if (hasEmulator) {
            // Remove emulator (but keep at least one? or allow empty = disabled?)
            return new Map(prev).set(systemId, current.filter(e => e !== emulatorId));
        } else {
            // Add emulator
            return new Map(prev).set(systemId, [...current, emulatorId]);
        }
    });
}
```

### Interaction Details

1. **Enabling a system**: Click "Enable System" → adds default emulator to list
2. **Adding emulator**: Check an unchecked emulator → adds to list
3. **Removing emulator**: Uncheck a checked emulator → removes from list
4. **Disabling system**: Click "Disable System" OR uncheck all emulators → removes system from config
5. **Version selection**: Dropdown per emulator row, changes stored in `emulatorVersions`

## Core Changes

### Registry

No changes needed. Registry already:
- Stores all emulators and their supported systems
- Provides `GetEmulatorsForSystem(SystemID) []Emulator`
- Provides `GetDefaultEmulator(SystemID) Emulator`

### Apply Process

Current flow:
```
for each system in config.Systems:
    emulator = system.Emulator
    install(emulator)
    configure(emulator, system)
    createDesktopFile(system, emulator)
```

New flow:
```
enabledEmulators = config.GetEnabledEmulators()  // Deduplicated

for each emulator in enabledEmulators:
    install(emulator)

    for each system in emulator.Systems:
        configure(emulator, system)
        createDesktopFile(system, emulator)
```

Key changes:
1. **Deduplication**: Dolphin installed once even if enabled for GC+Wii
2. **Desktop files**: One per (system, emulator) pair
3. **Config generation**: Called once per (emulator, system) pair

### Desktop File Generation

Current:
```
~/.local/share/applications/kyaraben-psx.desktop
    Name=PlayStation (Kyaraben)
    Exec=/path/to/duckstation
```

New:
```
~/.local/share/applications/kyaraben-duckstation-psx.desktop
    Name=PlayStation (DuckStation)
    Exec=/path/to/duckstation

~/.local/share/applications/kyaraben-retroarch_mednafen_psx-psx.desktop
    Name=PlayStation (Mednafen)
    Exec=/path/to/retroarch -L mednafen_psx
```

Naming convention: `kyaraben-{emulator_safe}-{system}.desktop`
- `emulator_safe`: EmulatorID with `:` replaced by `_`

### Manifest

No structural changes. Manifest already tracks emulators:

```toml
[emulators]
duckstation = { version = "v0.1-10655", nix_path = "..." }
"retroarch:mednafen_psx" = { version = "1.19.1", nix_path = "..." }
```

Multiple emulators for same system just means more entries here.

## Storage

No changes needed. Current structure already supports multi-emulator:

```
UserStore/
├── roms/{system}/           # Shared: all emulators read from here
├── bios/{system}/           # Shared: all emulators read from here
├── saves/{system}/          # Shared: cross-emulator save compatibility
├── states/{emulator}/       # Isolated: emulator-specific savestates
├── screenshots/{emulator}/  # Isolated: emulator-specific screenshots
└── opaque/{emulator}/       # Isolated: emulator-managed directories
```

## Migration

Since kyaraben is unreleased, we can make breaking changes. However, for development convenience:

### Config Migration

```go
func migrateConfigV1ToV2(old OldConfig) KyarabenConfig {
    new := KyarabenConfig{
        Global:    old.Global,
        Systems:   make(map[SystemID][]EmulatorID),
        Emulators: make(map[EmulatorID]EmulatorConf),
        Sync:      old.Sync,
    }

    for systemID, systemConf := range old.Systems {
        emulatorID, version := parseEmulatorString(systemConf.Emulator)
        new.Systems[systemID] = []EmulatorID{emulatorID}

        if version != "" {
            new.Emulators[emulatorID] = EmulatorConf{Version: version}
        }
    }

    return new
}
```

### Version Detection

```toml
# Old format (v1) - no version field, has systems.{system}.emulator
[systems.psx]
emulator = "duckstation"

# New format (v2) - systems.{system} is array
[systems]
psx = ["duckstation"]
```

Detection: Check if `systems.psx` is a table (v1) or array (v2).
